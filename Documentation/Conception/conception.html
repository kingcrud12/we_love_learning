Conception Technique & Fonctionnelle - WLL

Modélisation de la base de données, des flux et interfaces du projet.

1. Introduction à la Conception

Ce document fige l'architecture fonctionnelle et base de données de la plateforme We Love Learning (WLL). Il agit comme le contrat principal entre les besoins métiers de l'analyse préalable et le développement logiciel à venir.

Il regroupe :

  • La conception de données : MCD, MLD, MPD (MySQL).
  • La modélisation comportementale : Diagrammes de Cas d'utilisation et de Séquence.
  • Le rendu visuel (Wireframes / Maquettes).

2. Modèle Conceptuel de Données (MCD)

L'objectif de ce MCD est de représenter les concepts métiers (Entités) et leurs liens dans la plateforme de formation.

Concepts Métiers Identifiés

  • Utilisateur (Apprenant, Formateur, Admin) : L'acteur central de la plateforme.
  • Cours : Le conteneur principal du savoir, appartenant à un formateur.
  • Module & Leçon : La subdivision granulaire d'un cours.
  • Inscription : Le lien entre un Apprenant, un Cours et sa progression (pour le tracking Qualiopi).
  • Communauté / Message : L'axe "Social Learning" attaché aux leçons.

3. Modèle Logique de Données (MLD)

Le modèle logique est la traduction technique du MCD. Les entités deviennent des tables (attributs typés `VARCHAR`, `INT`, `DATETIME`, etc.), et les relations se transforment en Clés Étrangères (FK) avec l'application stricte de règles métiers.

4. Modèle Physique de Données (MPD) - MySQL

Le MPD prend en compte les contraintes spécifiques du SGBD MySQL. Voici le script DDL (Data Definition Language) avec la création des tables, contraintes d'intégrité, et un jeu de données de test (Data Manipulation) pour valider l'insertion.

-- Création de la base de données
CREATE DATABASE IF NOT EXISTS wll_db;
USE wll_db;

-- 1. Table Utilisateur
CREATE TABLE utilisateur (
    id INT AUTO_INCREMENT PRIMARY KEY,
    nom VARCHAR(255) NOT NULL,
    email VARCHAR(255) NOT NULL UNIQUE,
    mot_de_passe VARCHAR(255) NOT NULL,
    role ENUM('APPRENANT', 'FORMATEUR', 'ADMIN') DEFAULT 'APPRENANT',
    date_creation DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 2. Table Cours
CREATE TABLE cours (
    id INT AUTO_INCREMENT PRIMARY KEY,
    formateur_id INT NOT NULL,
    titre VARCHAR(255) NOT NULL,
    description TEXT,
    prix DECIMAL(10, 2) DEFAULT 0.00,
    publie BOOLEAN DEFAULT FALSE,
    FOREIGN KEY (formateur_id) REFERENCES utilisateur(id) ON DELETE CASCADE
);

-- 3. Inscription (Table de liaison avec progression)
CREATE TABLE inscription (
    id INT AUTO_INCREMENT PRIMARY KEY,
    apprenant_id INT NOT NULL,
    cours_id INT NOT NULL,
    date_inscription DATETIME DEFAULT CURRENT_TIMESTAMP,
    progression_pourcent INT DEFAULT 0 CHECK (progression_pourcent BETWEEN 0 AND 100),
    FOREIGN KEY (apprenant_id) REFERENCES utilisateur(id) ON DELETE CASCADE,
    FOREIGN KEY (cours_id) REFERENCES cours(id) ON DELETE CASCADE,
    UNIQUE(apprenant_id, cours_id)
);
