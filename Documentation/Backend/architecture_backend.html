Architecture Backend - WLL

Spécifications de l'API REST avec Spring Boot 3

1. Stack Technique Choisie

Le backend agit comme le cœur de la plateforme de formation : il expose les données via une API REST pure et gère la logique métier complexe (Qualiopi, Droits).

  • Framework Principal : Spring Boot 3.x
  • Langage : Java 17+ (ou 21 LTS)
  • Persistance : Spring Data JPA (Hibernate) + Base de données MySQL 8.x
  • Sécurité : Spring Security + JWT (JSON Web Tokens)
  • Build Tool : Maven (ou Gradle)

2. Architecture en Couches (N-Tiers)

L'application respecte une séparation stricte des responsabilités pour garantir la maintenabilité et faciliter les tests unitaires.

3. Spécification des Endpoints API REST

L'API respecte les standards REST (verbes HTTP sémantiques, codes de retours standards).

Utilisateurs & Auth (/api/auth)

Méthode Endpoint Description Accès
POST /api/auth/register Création d'un compte (Apprenant ou Formateur) Public
POST /api/auth/login Authentification et retour du Token JWT Public

Cours & Catalogue (/api/cours)

Méthode Endpoint Description Accès
GET /api/cours Liste des cours publiés (Catalogue) Public / Apprenant
POST /api/cours Créer un nouveau cours Formateur uniquement
PUT /api/cours/{id} Mettre à jour les infos d'un cours Formateur (propriétaire)

Apprentissage & Progression (/api/progressions)

Méthode Endpoint Description Accès
POST /api/progressions/lecon/{id} Marque une leçon comme terminée + MAJ l'inscription Apprenant inscrit
GET /api/progressions/cours/{id} Récupère la jauge % de progression d'un cours Apprenant inscrit

4. Sécurité & Stratégie JWT

La sécurité est un enjeu majeur, notamment pour la protection des vidéos des formateurs et l'accès aux attestations.

Flux d'Authentification (Stateless)

  1. Le client Angular envoie les credentials (Email/MDP) au /api/auth/login.
  2. Spring Security (via le AuthenticationManager) vérifie le hash avec BCrypt.
  3. Le serveur génère un JWT Token signé avec une clé secrète (contenant l'ID et les rôles de l'utilisateur), valide par exemple 24h.
  4. Le client stocke le token (LocalStorage/SessionStorage) et l'envoie dans le header Authorization: Bearer <token> pour CHAQUE future requête.
  5. Un JwtAuthenticationFilter intercepte les requêtes backend, décode le token, et vérifie les droits (ex: @PreAuthorize("hasRole('FORMATEUR')")).
// Exemple Configuration Spring Security (Sécurité Configurer)
@Bean
public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
    http.csrf(csrf -> csrf.disable())
        .sessionManagement(session -> session.sessionCreationPolicy(SessionCreationPolicy.STATELESS))
        .authorizeHttpRequests(auth -> 
            auth.requestMatchers("/api/auth/**").permitAll()
                .requestMatchers("/api/admin/**").hasRole("ADMIN")
                .requestMatchers(HttpMethod.POST, "/api/cours/**").hasRole("FORMATEUR")
                .anyRequest().authenticated()
        );
    http.addFilterBefore(jwtAuthenticationFilter, UsernamePasswordAuthenticationFilter.class);
    return http.build();
}

5. Mapping ORM (JPA) Typique

Les entités conçues dans le MLD/MPD sont directement mappées en classes Java annotées.

@Entity
@Table(name = "cours")
public class Cours {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(nullable = false)
    private String titre;

    // Mapping de la relation (Un Formateur crée plusieurs cours)
    @ManyToOne(fetch = FetchType.LAZY)
    @JoinColumn(name = "formateur_id", nullable = false)
    private Utilisateur formateur;

    // Un cours a plusieurs modules - Propagation en cascade
    @OneToMany(mappedBy = "cours", cascade = CascadeType.ALL, orphanRemoval = true)
    private List<Module> modules = new ArrayList<>();

    // Getters and Setters...
}
